Phase 1: Core Engine (Backend & Algorithm)
[ ] Setup Environment: Cài đặt Python, FastAPI, httpx (cho async requests).

[ ] Wiki API Wrapper: Viết hàm get_links(page_title) để lấy danh sách link từ 1 trang (Cần lọc bỏ các link rác như: Help:, Category:, User:, ISBN...).

[ ] Implement Bi-directional BFS: Xây dựng thuật toán tìm kiếm 2 đầu.

[ ] Caching Layer: Tích hợp Redis hoặc In-memory caching (LRU Cache) để lưu kết quả các trang đã tìm, tránh gọi lại Wikipedia nhiều lần.

[ ] API Endpoint: Tạo API POST /find-path nhận vào {source, target} và trả về danh sách [Person A, Person B, ..., Target].

Phase 2: Frontend & Visualization
[ ] UI Setup: Dựng khung nhập liệu (2 input box) và nút "Find Connection".

[ ] State Management: Xử lý trạng thái Loading (vì tìm kiếm sẽ mất vài giây đến phút).

[ ] Result Display: Hiển thị chuỗi kết quả dạng Step-by-Step.

[ ] Graph Viz (Nâng cao): Sử dụng thư viện react-force-graph hoặc d3.js để vẽ mối nối trực quan.

Phase 3: Optimization & Deploy
[ ] Rate Limiting Handler: Xử lý lỗi khi Wikipedia chặn request (429 Too Many Requests) bằng cơ chế Backoff/Retry.

[ ] Dockerize: Đóng gói ứng dụng để deploy dễ dàng.
